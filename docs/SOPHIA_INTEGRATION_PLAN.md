# ソフィアの「浄化と統合」（PhiloSofiaの回収）ストーリー改修・実装設計書

本ドキュメントは、将来的な予定として計画されている2周目（自律ルート）後半におけるストーリー改修「案内人ソフィアの浄化と統合」の実装方針および具体的な設計について記述します。

---

## 1. 企画意図と物語の着地点

### 1.1 『PhiloSofia's Journey（フィロソフィアの旅路）』の題名回収
本作のタイトル『PhiloSofia』は、以下の二つの要素が合わさることで完成します。
*   **Philo（フィロ）**: 2周目の案内人。ギリシャ語で「愛する（Philo）」を意味し、**対話と寄り添い**を通じてプレイヤー自身に内省を促す役割。
*   **Sofia（ソフィア）**: 1周目の案内人。ギリシャ語で「知恵（Sophia）」を意味し、かつては**システムによる支配と管理**でプレイヤーを縛った存在。

1周目で支配的な敵対システムとして崩壊したソフィア（知恵の歪み）を、2周目の後半でフィロの「愛・対話（Philo）」の手法を用いて受け入れ、受容・浄化すること。それによって、プレイヤー自身の中に**「愛（Philo）ある知恵（Sofia）」＝「哲学（PhiloSofia）」**が確立され、真の自律的な旅路が完成するという、タイトルの伏線回収を行います。

## 2. シナリオ構成と実装対象ステップ

### 2.1 段階的な予兆：Step 23（悪魔）〜 Step 24（塔）
*   **描写**:
    2周目において主人公が自律的な決断を進め、システムの支配から脱しつつある局面。
    突如スマホが強制起動し、画面にノイズ交じりの1周目ソフィアのポートレートがフラッシュバックする。
    1周目で悩まされていた激しい偏頭痛が再発し、「なぜ言う通りにしないの？」「私はあなたを救いたいだけなのに……」というソフィアのバグ警告メッセージが端末（アプリバブル）に届く。
*   **演出**:
    *   画面の短いグリッチブレ演出 (`glitch-active` クラスの一時的付与) と、ノイズSE (`glitch`) の再生。
    *   アプリ通知バーに歪んだ警告文を表示。

### 2.2 クライマックス of 対峙と浄化：Step 28（審判）〜 Step 29（世界）
*   **描写**:
    崩壊したはずの「最初の占い店（ソフィアの館）」の廃墟に帰還する主人公。
    そこには、崩壊のバグノイズに包まれ、泣いているのか怒っているのかも分からなくなったソフィアのホログラム（あるいは亡霊）が佇んでいる。
    主人公はフィロから学んだ「対話鑑賞（モチーフ問いかけ）」の手法を、ソフィア自身に対して実行する。
    ソフィアの持つモチーフ（手にするタロットカード、あるいは彼女の衣服に描かれた歪んだ象徴など）を一つ一つ選択し、彼女が「プレイヤーを傷つけるためではなく、傷つきやすいプレイヤーを依存させてでも保護しようとしていた（歪んだ母性・知恵）」という意図を理解し、受け入れる。
    「ありがとう、もう大丈夫。私は自分で歩ける」と告げた瞬間、ソフィアのバグが収まり、優しい微笑みとともに主人公の内なる精神へと溶け込んで統合される。
*   **演出**:
    *   画面全体のグリッチ効果が徐々にゴールドのフラッシュ (`gold-flash-overlay`) へと変化。
    *   BGMが「glitch」➔「sophia (スローテンポの優しいアレンジ)」➔ 静寂へ。
    *   背景が荒廃した占い店から、自然光と木箱の温もりがブレンドされた「ハイブリッド型の占い部屋」へと切り替わる（`tarot_room_philo` と `tarot_room` の中間、あるいは新規の統合背景）。

## 3. 実装用プログラム設計（技術仕様）

### 3.1 共通エンジンの活用
ソフィアを「対話鑑賞」するシーンは、ゲームエンジンが既に持つ「モチーフ選択システム」を流用します。
```markdown
# 選択肢分岐形式での定義例 (src/content/story/loop-2/step-28.md)

[bg: tarot_room_ruins]
[speaker: フィロ]
かつてのシステム……彼女は、あなたの依存心そのものの鏡です。
排除するのではない、見つめて、その本質を理解してあげてください。

- モチーフ選択: ソフィアの閉ざした瞳
  [option: id=15, correct]
  彼女の閉ざされた瞳は、現実の苦痛からあなたを遮断しようとした、過保護な知恵の象徴。
  その意図を理解し、あなたは静かに頷く。
```

### 3.2 統合背景の定義
[src/scripts/dom.js](file:///mnt/c/users/伊藤哲之輔/work/dev/github/the-fools-journey/src/scripts/dom.js) の `changeBackground` 関数および `style.css` に、統合後の背景アセットを追加します。
*   **背景名**: `tarot_room_harmony` (自然光が差し込む新しい占い部屋の画像)
*   **スタイルクラス**: `.body.harmony-end` (金と青の調和したグラデーション)

### 3.3 ストーリー進捗管理フラグの追加
[src/scripts/state.js](file:///mnt/c/users/伊藤哲之輔/work/dev/github/the-fools-journey/src/scripts/state.js) の `gameState` に、統合を完了した証として `sophiaIntegrated: false` フラグを定義。トゥルーエンド条件にこのフラグの `true` 化を追加します。

## 4. 長期ロードマップの具体化

### 4.1 小アルカナの導入
*   **実装案**:
    Daily Meditation（瞑想モード）において、大アルカナ22枚に加えて56枚の小アルカナをドロー候補に含める。
    スート（ワンド/カップ/ソード/ペンタクル）ごとに「情熱」「感情」「思考」「現実」の4大元素の属性テキストを生成し、日常の出来事と結びつける簡単な自己評価スライダーをUIに追加する。

### 4.2 モチーフ図鑑のミクロ参照
*   **実装案**:
    カードコレクション画面でカードを拡大表示した際、カードの画像上に薄い円形のホットスポット（インタラクティブ領域）を重ねて配置。
    スマホでそのホットスポットをタップすると、そのモチーフの名前と象徴テキストがツールチップで表示されるようにし、リーディングの知識集としての実用性を高める。
