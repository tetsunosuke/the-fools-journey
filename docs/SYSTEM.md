# 「プロジェクト・フール」システム仕様書（SYSTEM.md）

本仕様書は、メタ構造・日常侵食型ノベルアドベンチャーゲーム「プロジェクト・フール」のシステム、技術構成、およびスマートフォン向け最適化の設計仕様について記述します。

---

## 1. ターゲットプラットフォームと動作環境

本作は、手軽にアクセスして「日常が侵食される」体験を演出するため、**スマートフォンのWebブラウザを最優先のメインターゲット**として開発します。

*   **動作ターゲット**:
    *   スマートフォン（iOS/Android）の標準ブラウザ（Safari, Google Chrome）
    *   PCブラウザ（Safari, Chrome, Edge, Firefox）でのレスポンシブ表示も完全対応
*   **配信形態**: 静的HTMLホスティング（GitHub Pages, Netlify, Vercel等）によるWeb公開

---

## 2. 技術スタック

メンテナンス性、軽量化、およびメタギミックの自由度を両立するため、以下のスタックを使用します。

*   **基盤フレームワーク**: **Astro (v4.x)**
    *   高速な静的サイトジェネレート（SSG）とコンポーネント管理
    *   Viteによる快適なローカル開発環境（HMR対応）
*   **ユーザーインターフェース**: **HTML5 & Vanilla CSS**
    *   高速描画と滑らかなアニメーション制御のためのCSS VariablesおよびKeyframeアニメーションの多用
    *   TailwindCSS等のユーティリティは使用せず、柔軟なグリッチ表現や3DフリップのためにピュアなCSSでデザイン
*   **ゲームロジック**: **Vanilla JavaScript (ES6+)**
    *   ブラウザ環境そのものをメタ的に利用するため、コードの可読性をある程度保ち、過度な難読化を避けた設計
*   **画像アセット**:
    *   ライダー・ウェイト版（1909年）のパブリックドメインのタロットカード画像を事前にローカルダウンロードし、`public/images/cards/` 配下に静的アセットとして配置してゲーム内でロード（オフライン対応および読み込みの最適化）

---

## 3. スマートフォン向け最適化（タッチファースト設計）

スマホの狭い画面とタッチUIにおいて、快適な操作とメタ体験を両立するための独自システム仕様です。

### 3.1 独立した「マルチシーン（ビュー）」による画面構成の明確化
*   ゲームのコンテキストや演出に合わせて、画面全体を4つの独立したビュー（Talk:占い店対面, Chat:アプリトーク画面, Celtic:ケルト十字儀式, Puzzle:意思決定パズル）にクラス制御で瞬時に切り替えるルーティングシステムを実装しています。
*   これにより、占い師の肖像画が不要なシーンで常駐することを防ぎ、スマートフォンの画面効率を最大化します。

### 3.2 縦画面（ポートレート）に特化したレスポンシブUI
*   CSSの Flexbox と CSS Grid を用い、スマートフォンの画面アスペクト比（縦長）に自動最適化します。
*   チャットビューでは、メッセージが自動で下にスクロールされ、下部のインタラクションゾーンがキーボードや入力の阻害にならないよう最適配置されています。

### 3.3 ポインターイベントによるドラッグ＆ドロップ（カード破棄・石碑ギミック）
*   マウスとタッチパネル双方の入力を抽象化するため、**Pointer Events API**（`pointerdown`, `pointermove`, `pointerup`）を使用します。
*   これにより、スマホの画面スワイプやPCのドラッグ＆ドロップでも、ラグのない直感的な「カードをゴミ箱に捨てる」メタギミックを共通 of コードで動作させます。

### 3.4 スマホで完結する「象徴石碑ドラッグによる自律決定ギミック」
*   2周目の女教皇の試練において、単純なテキスト選択肢を廃止し、カードの周囲に散らばる「象徴の石碑（内省の鏡 / 他者拒絶）」をカードに直接ドラッグ＆ドロップして重ねることで、意思決定を行うゲームデザインを採用しています。
*   文字を入力させることなく、スマホのタッチ＆ドラッグ操作のみで「自分の意志で運命を動かした」という直感的なフィードバックと謎解き体験を両立させます。

## 4. ゲーム状態管理と永続化（セーブ）

プレイヤーの「ループ体験」を裏付けるための状態管理仕様です。

*   **メモリ上の状態（State）**:
    *   `currentLoop` (周回数: 1〜3+)
    *   `currentStep` (各周回内の進行シナリオインデックス)
    *   `isFateControlled` (運命支配フラグ: true=支配、false=自由)
*   **永続化仕様（セーブシステム）**:
    *   ブラウザの **LocalStorage** を使用して、プレイヤーがブラウザを閉じたりゲームを強制終了（リロード含む）したりしても、「これまでに何回ループしたか」「どのエンディングを見たか」の記憶を自動保持します。
    *   これにより、「ゲームをリセットしてやり直したはずなのに、占い師がプレイヤーの過去の行動を覚えている」といった認知ハッキング演出を実現します。

## 5. 最近のシステム拡張・変更点

### 5.1 話者タグパーサと動的スタイル制御（Talkビュー）
*   **状態変数 `activeDialogueSpeaker` の導入**:
    *   シナリオデータ（Markdownファイル）のページテキストの先頭に配置された `[speaker:話者名]` を正規表現で検知し、話者名を更新します。
    *   話者タグがないページでは `activeDialogueSpeaker` が維持され、冗長なタグ記述なしに会話が進行します。
*   **`.speaker-player` クラスによる動的スタイル変更**:
    *   `updateSpeakerVisibility` 関数において、話者名が「プレイヤー」の際に、対話ボックスへ `.speaker-player` クラスを着脱し、CSSで背景色や枠線を専用の配色に変更します。

### 5.2 空 `desc` 選択肢による自動進行（Chat / Talkビュー共通）
*   選択されたカードの `desc` が空であるかを判定し、空の場合は次のメッセージ描画完了後に直接 `advanceGame()` を呼び出して次のステップへ自動遷移させます。

### 5.3 デイリー・トランジション（暗転＆日付表示）演出
*   明示的な `[transition: X]` タグをMarkdown内に配置する設計へ移行しました。ステップロード時にこのタグを検知すると、画面全体を覆う黒いオーバーレイ（`day-transition-overlay`）の中央に日付とアルカナ名をフェードイン＆ズームで表示し、前ステップのテキスト残存を防ぐクリーンアップと同期して演出を実行します。

### 5.4 動的背景画像変更タグ（`[bg:画像名]`）
*   シナリオ内に `[bg:画像名]` を記述することで、対応する背景画像へ切り替えるパーサーを実装。タグ自体はパース時にテキストから自動削除されます。

### 5.5 チャット内選択肢結果（`desc`）のアプリ外表示化とインデックス不整合修正
*   チャット選択肢決定後、自動的に `talk` ビューに切り替えて結果をタイピング表示する仕様とし、インデックス不整合バグを解消しました。

### 5.6 ソウルカード算出時のサイバー数秘術アニメーション
*   数字スロットのブラー高速回転・拡大フラッシュ、光の尾を伴うクローン数字の射出飛行、着弾波紋スパーク、22以上警告バナー、衝突スライド合体、魔法陣超高速回転、およびゴールドフラッシュによる一連のトランジション演出を実装しました。

### 5.7 ケルト十字スプレッドの完全自動一括展開化（スマホ対応）
*   手動ドローを廃止し、450ms間隔で10枚のカードが自動配置されるアニメーション演出に統一。配置完了後にソフィアの解説テキストを流すように改善しました。

### 5.8 テキストウィンドウのレイアウト向上とスマホ向けレスパンシブ最適化
*   `.text-content` のスクロールバーを非表示化し、メディアクエリでスマホ時のケルト十字盤面高さを `52vh` に制限。ダイアログ領域が下に押しつぶされるのを防ぎ、十分な高さを確保しました。

### 5.9 シナリオMarkdownフォーマットの極限シンプル化
*   従来の複雑なYAMLフロントマター（フロントマター）定義を完全に撤廃しました。
*   物理的なフォルダ階層（`src/content/story/loop-1/step-00.md` 等）を採用し、Astroビルド時に `story.id` のファイルパスから自動で `loop` および `step` を抽出するパーサーを `index.astro` に実装しました。これにより、シナリオライターはYAMLメタデータの記述なしに純粋な脚本フォーマットのみでテキストを記述できるようになりました。

### 5.10 アプリバブル警告演出とシェイクアニメーションの導入
*   不正解カード（`correct: false`）を選択した際、これまでの画面上部への黒いシステム警告表示から、**画面右下の「アプリバブル」の上に「⚠️ SYSTEM WARNING」トーストをオーバーレイ表示し、同時にバブル本体が警告シェイク（震える）する演出**へとアップグレードしました。
*   アプリバブルがまだアンロックされていない導入ステップなどでは、自動的に従来の画面上部での警告表示へフォールバックされる安全な設計を実装しています。

### 5.11 占いシーンの選択肢（desc）統一と共通エンジン化
*   占いステップ（`step-12`）だけ進行フローが特殊であったため、通常の選択肢分岐フォーマット（「会話 ➔ 選択ボタン ➔ 選択結果のdesc」）に完全統一しました。
*   選択肢の `desc` の中に `[view: celtic]` を記述し、`processPageTags` でパースした段階でグローバルイベント `window.triggerViewChange` をキックしてビューを切り替える汎用的なフックを導入しました。
*   これにより、占いシーン専用の特殊なテキスト分割スライスロジック（`slice(0, 3)` など）をJSコード側から完全に撤廃し、全てのシナリオMarkdownが同一のゲームエンジン共通ルールで動作するようになりました。

### 5.12 人称の「あなた」統一とフレーバーテキストのクリーンアップ
*   アプリおよびソフィアがプレイヤーに語りかける人称表現をすべて「**あなた**」に完全統一しました。
*   これに伴い、実行時にプログラム側で行われていた機械的な置換処理（「あなた ➔ 君」）を削除し、定数ファイル（`constants.js`）およびストーリーMarkdownに最初から「あなた」と記述する設計へクリーンアップしました。
*   また、以前の変換時に混入していた `依存 of 心地よさ` ➔ `依存の心地よさ` 等の誤変換やタイポを完全に修正しました。

## 6. 解決済みのシステム不具合

### 6.1 ケルト十字スプレッドにおけるクリック進行不能バグ 【修正完了】
*   `highlightSlotGuide` 関数の未定義によるスクリプトの停止、および絶対配置されたカードが画面全体の基準にずれる問題を修正しました。

### 6.2 間違った選択肢（correct: false）選択時にゲームが進行してしまう不具合 【修正完了】
*   不正解の選択肢を選んだ場合は、警告演出を発生させた上で即座に早期リターンさせ、正しい選択肢を選び直す必要があるように制御を実装しました。
